Data Integration

Reliable data ingestion platform for analysis-ready data.

Quick Glance!

   Whether you're planning to build your data warehouse from scratch or looking for data ingestion or data automation, the DI platform enables you to integrate, source, and manage data pipelines over the cloud directly from your browser. With an array of features, DI can now automate and monitor your batch jobs, providing greater functionality and seamless user experience.

The tool aims to get your data into your warehouses faster than any traditional solutions.

   Helping Teams to centralize data from disparate sources with fully managed ETL architecture that supports many cloud connectors.

Key Aspects

Features offered by the platform

  • Data Sources

    DI supports most of the connectors both cloud and on-prem. Be it structured flat files or semi-structured JSON, DI leverages distributed processing to move your data within minutes to your destination in the format you prefer. Please see the full documentation.

  • Improved Performance

    Efficient data transfers that can handle Big Data requirements at ease. Behind the scenes, DI optimizes the pipeline based on the size of your dataset and uses open-source distributed frameworks like Apache Spark and HDFS for data parallelism and fast processing.

  • User Interface

    DI's integration with your exisiting applications can help enhance UX, increase user satisfaction and improve usabilty & accessibilty for non-techincal business users.

  • Change data capture

    Design pipeline for batch, slowly changing dimensions or master data in minutes. DI abstracts away technical implementation details and automates as much as possible - for easy setup and can be operated continuously either for master or transactional data load.

  • Automate Schedule

    After initial run, DI replicates the process and provides an option to schedule run on a daily, weekly, or monthly basis. DI frees you to focus on insights and dashboards, instead of fixing data leaks or pipeline issues.

  • Metadata Search

    Metadata is incredibly useful if used rightly - but that, too, becomes a challenge as an enterprise grows in scope and scale. We automated the metadata search at your source and extend reusability for schema changes.

Benefits

Built with business users in mind, our tool allows data teams to concentrate on finding trends in data.

  • The tool offers inbuilt logging and troubleshooting capabilities directly from the front-end. The user can take intuitive actions on failures or exceptions and fix them without hurdles. Detailed execution charts and logs will provide the exact layer where the pipeline breaks, making it convenient for data teams to handle issues.

  • Trigger run all the existing jobs without making changes in technical specifications. Users can run the failed jobs that are active without re-creating a new one again. The execution history provides a view of the failed jobs and we can trigger them right away from the browser.

  • DI enabled for multi-user will allow access to multiple users at the same time to log in and build jobs simultaneously and deploy them in production. The users are created with a definitive set of access rules. They can sign-in by identifying and authenticating themselves with the created credentials.

  • Monitor execution times of big data batch jobs. We built the tool with efficiency and performance in mind, it keeps track of the elapsed time of the job at different layers like extraction, staging, or loading — that helps us optimize the flow by using various connectors for the same job.

  • DI captures the incremental record counts that enable us to keep track of records that were upserted (Inserts and Updates if existing) in each run. We use update timestamps and load timestamps to calculate after each run. For Master data, we automated the lineage key by creating another dimension record for all the change captures.

  • Automated schema migrations — We help teams save resources with connectors that can automatically adapt to schema changes from your source to target systems. We create mapping STMs for various disparate sources that can be easily modified as and when required.

How it works

  • Python

    We use python 3.6 making it more powerful and fast. Also, open up endless opportunities for third-party integrations with our tool.

  • Remote Engine

    The entire framework runs on cloud remote engine that makes the tool highly available ensuring greater flexibility and scalability.

  • AWS Cloud

    The DI hosted on AWS lets us iterate, experiment, and innovate quickly as the modern data world advances.

  • Apache Spark

    We rely on data processing capabilities of Spark to distribute data processing tasks across multiple clusters for fast processing
